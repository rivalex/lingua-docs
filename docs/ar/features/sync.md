# المزامنة ثنائية الاتجاه

يمكن لـ Lingua استيراد الترجمات من الملفات المحلية إلى قاعدة البيانات وتصديرها مرة أخرى — مما يمنحك أفضل ما في الأمرين: **تشغيل مدعوم بقاعدة البيانات** و**التحكم في الإصدار المستند إلى الملفات**.

## الاتجاهان

```
lang/en/*.php        ──┐
lang/en.json           │  lingua:sync-to-database ──►  language_lines (DB)
lang/it/*.php          │
lang/it.json           │
lang/vendor/…          │  ◄── lingua:sync-to-local
                      ─┘
```

## المزامنة إلى قاعدة البيانات

يستورد جميع ملفات الترجمة من `lang/` (ومجلداتها الفرعية) إلى جدول `language_lines`.

```bash
php artisan lingua:sync-to-database
```

### ما يُستورد

- `lang/{locale}/*.php` — ملفات ترجمة PHP القياسية
- `lang/{locale}.json` — ملفات ترجمة JSON
- `lang/vendor/{package}/{locale}/*.php` — ترجمات حزم الطرف الثالث

### سلوك Upsert

تستخدم Lingua `updateOrCreate` مع تطابق `group` + `key`. هذا يعني:
- **المفاتيح الجديدة** تُدرج
- **المفاتيح الموجودة** يُدمَج JSON لـ `text` — قيم اللغات التي عدّلتها في الواجهة **تُحافظ عليها**
- **اكتشاف النوع** يعمل على القيمة لتحديد `text` / `html` / `markdown`

### الاكتشاف التلقائي للنوع

| القاعدة | النوع المُعيَّن |
|---|---|
| النص يحتوي على وسوم HTML (`<…>`) | `html` |
| النص يُحلَّل كـ Markdown (عناوين، قوائم، إلخ) | `markdown` |
| لا شيء من هذا | `text` |

::: tip
اكتشاف النوع محافظ — يُعيِّن `html` أو `markdown` فقط عندما يتطابق المحتوى بوضوح. النصوص العادية تحصل دائمًا على `text`. يمكنك تغيير النوع يدويًا عبر نافذة التعديل.
:::

### عبر الـ Facade

```php
use Rivalex\Lingua\Facades\Lingua;

Lingua::syncToDatabase();
```

::: warning
يستدعي الـ Facade `Translation::syncToDatabase()` داخليًا، وهو استدعاء ثابت. في أوامر Artisan ومكونات Livewire، يُستخدم `app(Translation::class)->syncToDatabase()` بدلًا من ذلك حتى يتمكن Mockery من اعتراضه في الاختبارات.
:::

## المزامنة إلى المحلي

يُصدِّر جميع الترجمات من `language_lines` مرة أخرى إلى ملفات PHP وJSON في `lang/`.

```bash
php artisan lingua:sync-to-local
```

### ما يُصدَّر

- جميع الترجمات غير الخارجية → `lang/{locale}/{group}.php`
- مفاتيح مجموعة JSON (`single`) → `lang/{locale}.json`
- ترجمات الحزم الخارجية → `lang/vendor/{vendor}/{locale}/{group}.php`

### حالات الاستخدام

- **التحكم في الإصدار** — قيّد الملفات المُصدَّرة لتتبع تغييرات الترجمة عبر الزمن
- **خطوط النشر** — صدِّر قبل النشر إذا كانت الأدوات الأدنى تتوقع ترجمات مستندة إلى الملفات
- **النسخ الاحتياطية** — أنشئ لقطة في نقطة زمنية لجميع الترجمات
- **أدوات أخرى** — صدِّر للاستخدام في خدمة إدارة الترجمات أو مستورد CSV

### عبر الـ Facade

```php
Lingua::syncToLocal();
```

## التحديث من Laravel Lang

يسحب أحدث نصوص الترجمة من نظام `laravel-lang` ويزامنها مع قاعدة البيانات. مفيد بعد ترقية Laravel أو إضافة حزمة جديدة تشحن بترجمات.

```bash
php artisan lingua:update-lang
```

يشغِّل هذا `lang:update` (من `laravel-lang/common`) متبوعًا بـ `lingua:sync-to-database`.

## سير عمل المزامنة الآلية

### عند النشر

أضف خطوة ما بعد النشر للحفاظ على تزامن قاعدة البيانات مع ملفات اللغة المُودَعة:

```bash
# In your deploy script or CI/CD pipeline
php artisan lingua:sync-to-database
php artisan optimize:clear
```

### عبر المجدول

إذا كان فريق الترجمة يُعدِّل الملفات مباشرة (بدلًا من الواجهة)، جدوِل مزامنة دورية:

```php
// routes/console.php
Schedule::command('lingua:sync-to-database')->hourly();
```

### عند تثبيت حزمة

عندما تقوم بـ `composer require` لحزمة جديدة تشحن بترجمات، شغِّل:

```bash
php artisan lingua:update-lang
```

هذا يلتقط أي نصوص جديدة من الحزمة المثبتة.

## نصائح وتحذيرات

::: tip اجعل قاعدة البيانات مصدر الحقيقة
تعامل مع قاعدة البيانات كمصدر أساسي. زامن إلى المحلي فقط عندما تحتاج إلى ملفات (التحكم في الإصدار، النشر، إلخ). تجنب تعديل الملفات المحلية مباشرة بينما قاعدة البيانات قيد الاستخدام — المزامنة التالية إلى قاعدة البيانات ستستبدل تعديلاتك إذا كانت المفاتيح موجودة بالفعل.
:::

::: warning ملفات اللغة وقاعدة البيانات غير متزامنة
إذا أضفت ملفات PHP للغة جديدة يدويًا دون تشغيل `lingua:sync-to-database`، المفاتيح الجديدة ستكون متاحة فقط عبر محمّل الملفات (أولوية أقل من قاعدة البيانات). شغِّل المزامنة لاستيرادها بشكل صحيح.
:::

::: tip الرحلة الكاملة ذهابًا وإيابًا
طريقة آمنة لإعادة تنظيم الترجمات:
1. `lingua:sync-to-local` — صدِّر كل شيء
2. عدِّل الملفات على القرص
3. `lingua:sync-to-database` — أعِد الاستيراد
:::
