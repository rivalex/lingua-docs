# ما هو Lingua؟

Lingua هي **حزمة Laravel** تستبدل نظام الترجمة المستند إلى الملفات الافتراضي بنظام مدعوم بالكامل بقاعدة البيانات، مكتمل بواجهة إدارة تفاعلية وأنيقة مبنية على [Livewire 4](https://livewire.laravel.com) و[Flux 2](https://fluxui.dev).

## المشكلة التي تحلها

ترجمات Laravel القياسية مخزّنة في ملفات PHP وJSON داخل `lang/`. يعمل هذا بشكل جيد للمشاريع الصغيرة، لكنه يُسبِّب احتكاكًا مع نمو التطبيق:

- **تحديث الترجمة يتطلب نشرًا** — حتى لإصلاح خطأ إملائي بسيط.
- **أعضاء الفريق غير التقنيين لا يمكنهم تعديل الترجمات** — يحتاج المحررون إلى Git ومراجعة الكود.
- **تتبع اكتمال الترجمة يدوي** — يجب مقارنة الملفات لإيجاد الثغرات.
- **دعم لغات متعددة يُعقِّد قاعدة الكود** — عشرات الملفات مبعثرة في مجلدات متعددة.

تخزّن Lingua كل ترجمة في قاعدة البيانات، في عمود JSON واحد لكل صف، وتوفر واجهة Livewire حيث يمكن لأي مستخدم مخوَّل إدارة اللغات والنصوص في الوقت الفعلي.

## كيف تعمل باختصار

```
┌─────────────────────────────────────────────────────────┐
│                   Laravel Application                   │
│                                                         │
│  lang/en/messages.php  ──┐                              │
│  lang/fr/messages.php    │  lingua:sync-to-database     │
│  lang/en.json            ├─────────────────────────────►│
│  lang/vendor/…           │                              │
│                         ─┘   language_lines (DB)        │
│                              ┌──────────────────────┐   │
│  LinguaMiddleware  ◄──────── │ group │ key │ text   │   │
│  app()->setLocale()          │ auth  │ … │ {"en":…} │   │
│                              └──────────────────────┘   │
│  __('auth.failed')  ───────────────────────────────────►│
│  (DB takes precedence over files)                       │
└─────────────────────────────────────────────────────────┘
```

في وقت التشغيل، تسجّل Lingua `LinguaManager` مخصصًا كمحمّل ترجمة لـ Laravel. يدمج الترجمات المستندة إلى الملفات وترجمات قاعدة البيانات — **مدخلات قاعدة البيانات تحظى دائمًا بالأولوية** — لذا يمكنك تجاوز أي نص دون لمس ملفات المصدر.

## المفاهيم الأساسية

| المفهوم | الوصف |
|---|---|
| **اللغة** | لغة مثبتة مع بيانات وصفية (الاسم، الاسم الأصلي، الاتجاه، ترتيب الفرز، علامة الافتراضي) |
| **الترجمة** | صف في `language_lines` يحتوي على `group` و`key` و`type` وعمود JSON لـ `text` يحتوي على جميع قيم اللغات |
| **نوع الترجمة** | `text` أو `html` أو `markdown` — يحدد المحرر المعروض في الواجهة |
| **ترجمة حزمة خارجية** | ترجمة تنتمي إلى حزمة طرف ثالث؛ محمية من الحذف غير المقصود |
| **اللغة الافتراضية** | اللغة الأساسية؛ حذف ترجمة اللغة الافتراضية يحذف السجل بالكامل |
| **المزامنة** | عملية استيراد الملفات المحلية → قاعدة البيانات (`sync-to-database`) أو تصدير قاعدة البيانات → ملفات (`sync-to-local`) |
