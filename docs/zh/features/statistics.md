# 翻译统计

统计页面实时展示所有已安装语言的翻译覆盖率。

```blade
<a href="{{ route('lingua.statistics') }}">翻译统计</a>
```

<Screenshot src="/screenshots/statistics-page.png" alt="翻译统计页面" caption="统计页面显示每种语言的覆盖率、组别明细和缺失键。" />

## 概览

页面顶部显示三个摘要计数器：

- **键总数** — 数据库中翻译字符串的总数
- **组总数** — 不同翻译组的数量
- **已安装语言** — 系统中注册的语言数量

您可以切换**包含 vendor 翻译**，以在所有统计中包含或排除包翻译。

## 按语言划分的覆盖率

<Screenshot src="/screenshots/statistics-coverage.png" alt="按语言覆盖率部分" caption="每个语言行显示进度条、百分比和缺失键数量。" />

对于每种已安装的语言，您可以看到：

| 列 | 说明 |
|---|---|
| 语言 | 语言的名称和国旗 |
| 覆盖率 | 带有完成百分比的进度条 |
| 已翻译 | 该语言环境有值的键数量 |
| 缺失 | 没有值的键数量 — 点击展开缺失键面板 |

**默认语言**以徽章高亮显示。由于默认语言环境驱动参考键集，它始终显示 100% 覆盖率。

## 缺失键下钻

点击任何语言行上的**缺失数量徽章**，可以内联展开缺失键面板。

<Screenshot src="/screenshots/statistics-missing.png" alt="已展开的缺失键面板" caption="缺失键面板显示组、键以及指向翻译编辑器的直接链接。" />

面板中每一行显示：

- **组** — 翻译组（例如 `validation`、`auth`）
- **键** — 翻译键（例如 `required`、`failed`）
- **翻译 →** 链接 — 打开预先过滤到该语言环境的翻译页面

再次点击同一语言行可折叠面板。

## 按组明细

<Screenshot src="/screenshots/statistics-breakdown.png" alt="按组明细表" caption="明细表显示每个组每种语言环境的已翻译键数量。" />

明细表列出每个翻译组以及每种已安装语言的已翻译键数量。用它来识别哪些组已完全翻译，哪些需要关注。

## Vendor 翻译切换

默认情况下，vendor 翻译（来自 `laravel/framework` 等包）被排除在统计之外。切换**包含 vendor 翻译**以将其包含在内。

::: info
Vendor 翻译通常由包作者在上游维护。如果您没有添加特定语言环境的覆盖，将其包含在统计中可能会降低您的表观覆盖率。
:::

::: tip 高效工作
按缺失数量排序，优先处理翻译工作。列表顶部缺失键最多的语言需要最多的关注。
:::
