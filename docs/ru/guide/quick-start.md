# Быстрый старт

Это руководство проведёт вас от свежей установки Lingua до полноценного многоязычного приложения за несколько минут.

## 1. Установка и заполнение данными

```bash
composer require rivalex/lingua
php artisan lingua:install
```

Английский язык теперь является языком по умолчанию, и все строки переводов Laravel/пакетов импортированы в базу данных.

## 2. Добавление второго языка

```bash
php artisan lingua:add fr
```

Эта команда:
- Скачивает файлы французских переводов через Laravel Lang
- Создаёт запись `Language` в базе данных
- Синхронизирует все новые строки в `language_lines`

Повторите для нужного количества локалей:

```bash
php artisan lingua:add it es de pt_BR ar
```

## 3. Добавление переключателя языков в layout

Откройте ваш основной Blade-layout (например, `resources/views/layouts/app.blade.php`) и:

**а) Установите `lang` и `dir` на теге `<html>`:**

```blade
<html lang="{{ app()->getLocale() }}" dir="{{ Lingua::getDirection() }}">
```

**б) Встройте переключатель языков туда, где это подходит вашему дизайну:**

```blade
{{-- Как группа в сайдбаре (по умолчанию) --}}
<livewire:lingua::language-selector />

{{-- Как выпадающий список в навбаре --}}
<livewire:lingua::language-selector mode="dropdown" />
```

## 4. Использование переводов в приложении

Lingua прозрачна - используйте стандартные хелперы Laravel как обычно:

```blade
{{ __('messages.welcome') }}
@lang('auth.failed')
```

```php
trans('validation.required')
__('pagination.next')
```

Кастомный `LinguaManager` автоматически накладывает database-переводы поверх файловых. Изменения в коде не нужны.

## 5. Перевод через интерфейс

Перейдите на `/lingua/translations`, чтобы увидеть все строки переводов. Для каждого языка:

1. Используйте переключатель локалей (вверху справа) для выбора целевого языка
2. Нажмите на любую строку, чтобы редактировать значение прямо в таблице
3. Используйте **Только непереведённые**, чтобы сосредоточиться на непереведённых строках
4. Для типов HTML или Markdown rich-text редактор активируется автоматически

## 6. Синхронизация обратно в файлы (опционально)

Если вам нужны файлы переводов на диске (для версионного контроля, CI/CD или других инструментов):

```bash
php artisan lingua:sync-to-local
```

Это экспортирует все переводы из БД обратно в `lang/` в правильном формате PHP/JSON.

## Распространённые сценарии

### Создание нового ключа программно

```php
use Rivalex\Lingua\Facades\Lingua;

// Создание перевода в базе данных для локали по умолчанию
// (обычно это делается через интерфейс, но можно и скриптом)
Translation::create([
    'group' => 'emails',
    'key'   => 'welcome_subject',
    'type'  => 'text',
    'text'  => ['en' => 'Welcome to our platform!'],
]);

// Позже добавьте переводы для других локалей:
Lingua::setTranslation('welcome_subject', 'Bienvenue sur notre plateforme !', 'fr');
Lingua::setTranslation('welcome_subject', 'Benvenuto nella nostra piattaforma!', 'it');
```

### Проверка полноты переводов

```php
$stats = Lingua::getLocaleStats('fr');
// [
//   'total'      => 1240,
//   'translated' => 980,
//   'missing'    => 260,
//   'percentage' => 79.03,
// ]
```

### Переключение локали программно

```php
// В контроллере, middleware или сервисе
app()->setLocale('fr');
session()->put(config('lingua.session_variable'), 'fr');
```

::: tip
Компонент `LanguageSelector` автоматически обрабатывает переключение локали для конечных пользователей. Описанный выше ручной подход полезен в консольных командах или задачах.
:::

### Экспорт только определённых групп

Если вы хотите экспортировать только часть переводов в файлы, сначала синхронизируйтесь в локальные файлы, затем удалите ненужные группы из `lang/` - база данных всегда является источником истины во время выполнения.
