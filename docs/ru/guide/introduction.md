# Что такое Lingua?

Lingua — это **пакет для Laravel**, который заменяет стандартную файловую систему переводов на полностью основанную на базе данных, с красивым реактивным интерфейсом управления, построенным на [Livewire 4](https://livewire.laravel.com) и [Flux 2](https://fluxui.dev).

## Какую проблему он решает

Стандартные переводы Laravel хранятся в PHP и JSON файлах в директории `lang/`. Это хорошо работает для небольших проектов, но создаёт сложности по мере роста приложения:

- **Обновление перевода требует деплоя** — даже для исправления опечатки.
- **Нетехнические члены команды не могут редактировать переводы** — редакторам нужен доступ к Git и прохождение code review.
- **Отслеживание полноты переводов — ручная работа** — приходится сравнивать файлы, чтобы найти пробелы.
- **Поддержка множества локалей загромождает кодовую базу** — десятки файлов, разбросанных по директориям.

Lingua хранит все переводы в базе данных в едином JSON-столбце на строку и предоставляет интерфейс на Livewire, где любой авторизованный пользователь может управлять языками и строками в режиме реального времени.

## Как это работает

```
┌─────────────────────────────────────────────────────────┐
│                   Laravel Application                   │
│                                                         │
│  lang/en/messages.php  ──┐                              │
│  lang/fr/messages.php    │  lingua:sync-to-database     │
│  lang/en.json            ├─────────────────────────────►│
│  lang/vendor/…           │                              │
│                         ─┘   language_lines (DB)        │
│                              ┌──────────────────────┐   │
│  LinguaMiddleware  ◄──────── │ group │ key │ text   │   │
│  app()->setLocale()          │ auth  │ … │ {"en":…} │   │
│                              └──────────────────────┘   │
│  __('auth.failed')  ───────────────────────────────────►│
│  (DB takes precedence over files)                       │
└─────────────────────────────────────────────────────────┘
```

Во время выполнения Lingua регистрирует собственный `LinguaManager` в качестве загрузчика переводов Laravel. Он объединяет файловые и database-переводы — **записи в базе данных всегда имеют приоритет** — поэтому вы можете переопределить любую строку, не трогая исходные файлы.

## Ключевые концепции

| Концепция | Описание |
|---|---|
| **Язык** | Установленная локаль с метаданными (название, нативное название, направление, порядок сортировки, флаг по умолчанию) |
| **Перевод** | Строка в `language_lines` с `group`, `key`, `type` и JSON-столбцом `text`, содержащим значения для всех локалей |
| **Тип перевода** | `text`, `html` или `markdown` — определяет, какой редактор отображается в интерфейсе |
| **Перевод пакета (vendor)** | Перевод, принадлежащий стороннему пакету; защищён от случайного удаления |
| **Локаль по умолчанию** | Основной язык; удаление перевода для локали по умолчанию удаляет всю запись |
| **Синхронизация** | Процесс импорта локальных файлов → БД (`sync-to-database`) или экспорта БД → файлы (`sync-to-local`) |
