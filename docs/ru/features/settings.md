# Настройки

Страница настроек позволяет настраивать поведение UI Lingua прямо из браузера — без редактирования файлов конфигурации или повторного развёртывания.

Перейдите на `/lingua/settings` или добавьте ссылку из вашей панели администратора:

```blade
<a href="{{ route('lingua.settings') }}">Настройки Lingua</a>
```

## Как работают настройки

Настройки хранятся в таблице базы данных `lingua_settings` как типизированные пары ключ/значение. При каждом запросе Lingua сначала читает из базы данных, затем обращается к `config/lingua.php`, затем к встроенным значениям по умолчанию.

**Цепочка приоритетов:**
1. Таблица DB `lingua_settings` (наивысший приоритет — задаётся через этот UI)
2. `config/lingua.php` (ваша опубликованная конфигурация)
3. Значения по умолчанию пакета (наименьший приоритет)

Вы можете использовать `config/lingua.php` как базовую линию и переопределять отдельные настройки для каждой среды через UI без изменения файлов.

## Режим селектора

Управляет тем, как компонент `<livewire:lingua::language-selector />` отображается для ваших конечных пользователей.

| Режим | Описание |
|---|---|
| `sidebar` | Отображается как сгруппированный раздел навигации (по умолчанию) |
| `modal` | Отображается как кнопка, открывающая полноэкранный модальный выбор языка |
| `dropdown` | Отображается как компактная кнопка с выпадающим списком |
| `headless` | Без встроенного отображения — вы реализуете UI самостоятельно |

::: tip Режим headless
При значении `headless` встроенный селектор ничего не отображает. Вместо него используйте `<livewire:lingua::headless-language-selector />` для создания полностью кастомного переключателя. См. [Headless-селектор](./language-selector#режим-headless) для полной документации.
:::

## Показывать иконки флагов

Включение или отключение отображения иконок национальных флагов рядом с названиями языков в селекторе. При отключении показывается только название языка.

Иконки флагов сопоставляются с кодом `regional` языка (например, `en_US` → 🇺🇸). Если региональный код не задан, флаг корректно скрывается.

## Программный доступ

Вы можете читать и записывать настройки на PHP с помощью модели `LinguaSetting`:

```php
use Rivalex\Lingua\Models\LinguaSetting;

// Чтение с fallback config()
$mode = LinguaSetting::get(
    LinguaSetting::KEY_SELECTOR_MODE,
    config('lingua.selector.mode', 'sidebar')
);

$showFlags = LinguaSetting::get(
    LinguaSetting::KEY_SHOW_FLAGS,
    config('lingua.selector.show_flags', true)
);

// Запись
LinguaSetting::set(LinguaSetting::KEY_SELECTOR_MODE, 'modal');
LinguaSetting::set(LinguaSetting::KEY_SHOW_FLAGS, false);
```

Доступные константы:

| Константа | Ключ | Тип |
|---|---|---|
| `LinguaSetting::KEY_SELECTOR_MODE` | `selector.mode` | `string` |
| `LinguaSetting::KEY_SHOW_FLAGS` | `selector.show_flags` | `bool` |

::: warning Требуется миграция
Таблица `lingua_settings` создаётся миграцией `create_lingua_settings_table`. Если вы обновились с версии 1.0.x, выполните `php artisan migrate` для её создания.
:::
