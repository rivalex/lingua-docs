# Управление языками

Страница «Языки» (`/lingua/languages`) — ваш центр управления всеми установленными локалями.

<Screenshot src="/lingua-docs/screenshots/languages-page.png" alt="Страница управления языками Lingua" caption="Страница языков — установленные локали со статистикой заполнения." />

## Добавление языка

### Через интерфейс

Нажмите **Add Language**, выберите любую из 70+ доступных локалей и подтвердите. Lingua выполнит:

1. Скачает языковые файлы из Laravel Lang
2. Создаст запись `Language` в базе данных
3. Синхронизирует все новые строки в `language_lines`
4. Обновит таблицу с новой локалью

<Screenshot src="/lingua-docs/screenshots/language-add-modal.png" alt="Модальное окно добавления языка" caption="Модальное окно добавления языка с поиском по локали." width="640px" :center="true"/>

### Из командной строки

```bash
php artisan lingua:add it
php artisan lingua:add pt_BR
php artisan lingua:add ar
```

### Программно

```php
use Rivalex\Lingua\Facades\Lingua;

// Установить языковые файлы (обёртка над lang:add)
Lingua::addLanguage('fr');

// Затем создать запись в БД + синхронизация (что Artisan-команда делает полностью)
// → используйте lingua:add для полного оркестрированного процесса
```

::: tip
Используйте `Lingua::notInstalled()`, чтобы получить список доступных, но ещё не установленных локалей:

```php
$available = Lingua::notInstalled(); // ['af', 'ar', 'az', …]
```
:::

## Удаление языка

Нажмите иконку корзины в строке любого неосновного языка. Модальное окно подтверждения предотвращает случайное удаление — нужно ввести название языка для подтверждения.

За кулисами операция удаления:
1. Удаляет языковые файлы через `lang:rm {locale} --force`
2. Удаляет все записи `{locale}` из JSON-столбца `language_lines.text`
3. Удаляет запись `Language`
4. Переупорядочивает значения `sort` для оставшихся языков

::: warning
**Язык по умолчанию нельзя удалить**. Сначала сделайте другой язык основным.
:::

```bash
# Из командной строки
php artisan lingua:remove fr
```

## Установка языка по умолчанию

Нажмите иконку звезды (⭐) в строке любого языка. Только один язык может быть основным одновременно. Изменение обёрнуто в транзакцию базы данных, чтобы предотвратить ситуацию, когда ни один язык не помечен как основной.

```php
// Программно
Lingua::setDefaultLocale('fr');

// Или через модель
$french = Language::where('code', 'fr')->first();
Language::setDefault($french);
```

::: warning Смена языка по умолчанию
Если вы устанавливаете новый язык по умолчанию, убедитесь, что переводы для этой локали хотя бы частично заполнены. Язык по умолчанию используется как запасной в редакторе интерфейса (левая колонка показывает значение по умолчанию в качестве справки).
:::

## Изменение порядка языков

Перетащите строки языков, чтобы управлять порядком их отображения во всём приложении — в виджете выбора языка, переключателе локалей на странице переводов и везде, где используется `Lingua::languages()`.

Порядок сортировки хранится в целочисленном столбце `sort` и переназначается последовательно после каждого перетаскивания.

## Статистика заполнения

В каждой строке языка отображается:

| Метрика | Описание |
|---|---|
| **% заполнения** | `переведено / всего * 100`, округлено до 2 знаков |
| **Отсутствует** | Количество строк без значения для данной локали |

Статистика вычисляется в момент запроса через подзапросы к таблице `language_lines`, поэтому всегда актуальна.

```php
// Получить статистику для конкретной локали
$stats = Lingua::getLocaleStats('fr');
// [
//   'total'      => 1240,
//   'translated' => 980,
//   'missing'    => 260,
//   'percentage' => 79.03,
// ]

// Или получить все языки со статистикой одним запросом
$languages = Lingua::languagesWithStatistics();
foreach ($languages as $lang) {
    echo "{$lang->name}: {$lang->completion_percentage}%";
}
```

## Элементы управления синхронизацией

На панели инструментов страницы «Языки» есть три кнопки синхронизации:

| Кнопка | Действие |
|---|---|
| **Sync to database** | Импортирует все локальные файлы `lang/` в `language_lines` |
| **Sync to local** | Экспортирует все переводы из БД обратно в файлы `lang/` |
| **Update via Laravel Lang** | Запускает `lang:update` для получения последних строк, затем синхронизирует в БД |

Все три операции выполняются **асинхронно** (атрибут Livewire `#[Async]`), поэтому интерфейс остаётся отзывчивым во время длительной синхронизации.
